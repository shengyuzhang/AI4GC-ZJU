# 实验室主页 · 学生编辑指南

这份指南教你**不用任何命令行、全程在 GitHub 网页上**更新自己在实验室主页上的内容：
更新自我介绍、新增一篇论文、发布一篇博客、更换头像。

你的每一次修改都会先变成一个 **Pull Request（PR，修改提议）**，由老师 review（审核）通过后才会更新到线上网站。

---

## 0. 准备工作（只需做一次）

1. 注册一个 [GitHub](https://github.com) 账号，把用户名发给老师（老师会给你主仓库的访问权限）。
2. 打开实验室**主仓库**地址（老师会发给你），点页面右上角的 **Fork**，把仓库复制一份到你自己的账号下。这份副本就是你的"工作区"。
3. 以后你所有的编辑都在**你自己 Fork 出来的那份仓库**里进行（网址形如 `github.com/你的用户名/仓库名`）；改完通过 **Pull Request** 提交回实验室主仓库，老师 review 通过后才会上线。

> 为什么要 Fork？这样你只能改自己的副本，物理上**不可能**误动到线上主仓库或别人的内容——更安全，老师也放心。
> 主仓库有更新后，到你 Fork 页面点 **Sync fork** 就能把最新内容同步过来。

## ⭐ 黄金法则

- **只修改属于你自己的内容**：你自己的成员文件夹、你自己的博客、你自己的论文。不要改别人的文件或网站代码。
- **每次只做一件事**：例如"更新简介"开一个 PR，"发博客"再开一个 PR，方便老师审核。
- **不确定就先提 PR**：提了 PR 不会马上上线，老师会先看。有疑问在 PR 里留言问即可。

## 内容放在哪里？

```
content/
├── team/{组}/{你的文件夹}/      ← 你的个人信息
│   ├── index.md                  ← 自我介绍、标签、邮箱、链接
│   ├── photo.jpg                 ← 你的头像
│   └── papers.bib                ← 你的论文库
└── blog/{文章文件夹}/
    ├── index.md                  ← 博客正文
    └── cover.png                 ← 博客配图（可选）
```

> 你的"文件夹"名字形如 `yurun-chen-2025-12551024`（姓名-入学年-学号）。找不到自己的文件夹就问老师。
> 所有内容的字段格式以仓库根目录的 [`content/guidance.md`](../content/guidance.md) 为准。

---

> 下面所有任务，都在**你自己 Fork 的那份仓库**（`github.com/你的用户名/...`）里操作。

1. 在你 Fork 的仓库页面，进入你的文件夹：`content` → `team` → `你的组` → `你的文件夹`，点开 `index.md`。
2. 点右上角的**铅笔图标 ✏️**（Edit this file）。
3. 文件分两部分：
   - 顶部两个 `---` 之间是**字段信息**（姓名、标签 `tags`、邮箱 `email`、外部链接 `links` 等）。
   - 下面是**正文**（你的自我介绍，用 Markdown 写）。改这里更新介绍。
4. 改完，点右上角 **Commit changes...**。
5. 在弹窗里**选中** "Create a new branch for this commit and start a pull request"（为这次修改新建分支并发起 PR），然后 **Propose changes**。
6. GitHub 会跳到发起 PR 的页面，并自动把 **base（目标）设为实验室主仓库的 `master`**。确认目标仓库无误后，填一句说明，点 **Create pull request**。完成！

**可选：在个人页展示 Blog / 站外渠道**

1. 在 `index.md` 正文末尾（或合适位置）加一行：`## @blog Lab Notes`（`Lab Notes` 可改成你喜欢的标题）。
2. 若要挂小红书 / 公众号 / X 等**长期主页**，在 frontmatter `links` 里加（注意 `kind: blog-channel`）：
   ```yaml
   - label: RedNote (小红书)
     href: https://www.xiaohongshu.com/user/profile/你的主页ID
     kind: blog-channel
     desc: 一两句小字说明，会显示在标题下方。
   ```
3. GitHub、Scholar 等社交链接用普通 `links`（不设 `blog-channel`），显示在页面顶部。已用 `blog-channel` 挂过的主页，不必在 Hero 再写一遍。

## 任务二：新增一篇论文

分两步——先把论文加进论文库，再把它列到你的主页上。

**第 1 步：在 `papers.bib` 里添加论文**
1. 进入你的文件夹，点开 `papers.bib`，点铅笔 ✏️。
2. 在文件里粘贴一条 BibTeX，**注意第一行花括号里那个 `引用键(cite key)` 要唯一**，例如下面的 `myNewPaper2026`：
   ```bibtex
   @article{myNewPaper2026,
     title={Your Paper Title Here},
     author={Your Name and Coauthor Name and Shengyu Zhang},
     year={2026},
     journal={arXiv},
     url={https://arxiv.org/abs/xxxx.xxxxx},
   }
   ```
   > 已发表会议/期刊用 `@inproceedings{...}` 或 `@article{...}`，并把 `journal=` 换成会议/期刊名；arXiv 预印本就写 `journal={arXiv}`。
3. 先**不要**提交，继续第 2 步——也可以分两个 PR，但放一起更方便。

**第 2 步：在 `index.md` 里把论文显示出来**
1. 打开你的 `index.md`，找到正文里的 `## @papers` 区块（如果没有，自己加一行 `## @papers Selected Papers`）。
2. 在下面**用列表加上刚才那条论文的引用键**：
   ```markdown
   ## @papers Selected Papers

   - myNewPaper2026
   - （你原有的其它论文键……）
   ```
3. 两个文件都改好后，按"任务一"的第 4~6 步，新建分支并发起 PR。

> 关键点：`papers.bib` 里的 `引用键` 必须和 `index.md` 里 `@papers` 下写的**完全一致**，否则会显示不出来（CI 检查也可能报错）。

## 任务三：发布一篇博客

1. 在 GitHub 仓库页面进入 `content/blog`。
2. 点 **Add file → Create new file**。
3. 在文件名输入框里输入：`你的文章英文短名/index.md`
   —— 注意中间的 `/`，它会自动帮你创建一个文件夹。例如 `my-cool-result-2026/index.md`。
4. 把下面模板粘进去并修改：
   ```markdown
   ---
   title: '你的博客标题'
   date: Jun 2026
   authorId: 你的文件夹名（例如 yurun-chen-2025-12551024）
   desc: >-
     一两句话的摘要，会显示在博客列表和卡片上。
   links:
     - kind: paper
       href: https://arxiv.org/abs/xxxx.xxxxx
     - kind: code
       href: https://github.com/你的用户名/仓库名
   tags:
     - 标签1
     - 标签2
   ---
   这里开始写正文，用 Markdown。

   ## 小标题

   正文段落……外部链接直接写在正文里即可。
   ```
   > `authorId` 填你的成员文件夹名；个人页须同时有 `## @blog …` 区块，文章才会出现在 **On-site** 分组。
   > 多个作者用 `authorIds:` 列表。`links` 可选，用于文章开头的 Paper / Code / 小红书 / 微信等资源按钮。
5. 点 **Commit changes...**，同样选择"新建分支并发起 PR"。

**给博客配图：** 提了 PR 后（或编辑时），用下面"任务四"的 github.dev 方式，把图片上传到**同一个文章文件夹**里，正文用 `![说明](图片文件名.png)` 引用。

## 任务四：更换头像 / 上传图片

网页铅笔不能上传图片，用 **github.dev**（网页版编辑器，免安装）：

1. 在仓库主页**按一下键盘的 `.`（句号）键**，页面会变成网页版 VS Code。
2. 左侧文件树找到你的文件夹，把图片**直接拖进去**（头像建议命名 `photo.jpg`，覆盖旧的即可）。
3. 如果换了头像文件名，记得到 `index.md` 把 `photo:` 那一行改成新文件名。
4. 左侧点"源代码管理"图标（带数字的分支图标）→ 在上方输入一句说明 → 点 **Commit & Push** → 选择新建分支。
5. 回到 GitHub 仓库页面，它会提示你 **Compare & pull request**，点它发起 PR。

## 提交之后会发生什么？

1. **自动检查**：PR 页面下方会出现一项 **Content & build check**。
   - ✅ 绿色 = 格式没问题。
   - ❌ 红色 = 有格式错误，点 "Details" 看提示，回到文件改正后再提交一次（PR 会自动更新）。
2. **老师审核**：检查通过后，老师会 review 你的改动，可能直接合并，也可能在 PR 里留言让你调整。
3. **自动上线**：老师点 Merge 后，网站会自动重新构建，几分钟后你的改动就出现在线上主页了。

## 常见问题

- **我看不到自己的文件夹？** → 找老师确认你的文件夹名/分组。
- **CI 一直红色？** → 多半是 YAML 缩进、引号、或 BibTeX 引用键不匹配。点 Details 看具体哪一行报错。
- **改错了想撤销？** → 直接关掉这个 PR（Close pull request）就行，不会影响线上。
- **想先看看改完长什么样？** → 如果配置了 Vercel 预览，PR 里会出现一个预览链接，点开就能看到你这次改动的效果。
